# StudentAge Editor Plus — 编辑器修复说明

> 一款用于《学生时代》(StudentAge) 的插件，修复游戏自带 MOD 编辑器的若干问题，让 mod 制作更加顺畅。
> 取代已废弃的旧插件 `StudentAgeModEditorFix`。

- **插件名**：StudentAge Editor Plus
- **运行前置**：需先安装 BepInEx（游戏 mod 通用前置框架）
- **适用对象**：mod 制作者。玩家游玩装有这些 mod 的存档时**无需**安装本插件（详见各节末尾说明）。

## 1. 场景事件触发类型补全（✅ 已修复）

### 问题
部分场景（如鹅城博物馆、黄金海滩、客运站等）的「进入/离开」事件类型，在编辑器的类型下拉框中无法选择，作者只能手动修改配置文件才能配置。

### 修复后
打开编辑器时，插件会自动补全这些缺失的选项。现在下拉框中可以直接选择「场景名(进入)」和「场景名(离开)」。

### 注意事项
- 新开发区、南城名苑、老城区这三个场景只能配置「进入」事件，「离开」因游戏本身的编号限制无法配置。
- 制作出的 mod 玩家可正常触发，**玩家无需安装本插件**。

## 2. 「场景」字段提示（✅ 已加）

### 问题
事件编辑器中的「场景」字段看起来像是"将事件绑定到某个场景"，但它实际上只是某些随机事件的筛选条件，并不能让事件在"进入场景时触发"，容易导致作者误填。

### 修复后
鼠标悬停在该字段上时会显示说明：此字段仅用于筛选，若要让事件在进入/离开某场景时触发，请改用上方的「类型」字段，选择「场景名(进入)」或「场景名(离开)」。

## 3. CG 编辑器：分组 / 编号可见（✅ 已修复）

### 问题
CG 编辑器中看不到「组别」和「编号」两个字段，作者无法整理和编排自己的 CG。

### 修复后
CG 编辑界面现已可填写「组别」和「编号」：
- **组别**：决定该 CG 归入收藏画廊的哪一分类（填负数时隐藏）；填 0 或 1 时该 CG 还可被选用为背景。
- **编号**：决定 CG 在画廊中的排列顺序（填大于 0 的数值时会显示为 001/002… 三位数格式）。

## 4. 图片预览（✅ 已修复）

### 问题
编辑器右侧原本完全不显示图片。受影响的编辑器包括：CG、背景、物品、书籍、珍贵记忆、行动、状态、企鹅空间动态、头像。

此外，更换新图或修改图片路径后点击「保存」，预览仍显示旧图，必须退出编辑器重新进入才能看到新图。

### 修复后
- 选中条目时，右侧会自动显示其图片预览，支持滚轮缩放、拖动平移、双击复位。
- 换图或点击保存后，预览会立即更新为新图，无需退出重进。

> 注：人物和结局使用各自专门的编辑器，游戏本身已带图片预览，因此未作改动。

## 5. 物品编辑器：持有上限可见（✅ 已修复）

### 问题
编辑器中没有「持有上限」字段，导致消耗品默认最多只能持有 1 个——玩家购买第 2 个会被截断，用完后归 0。游戏原版消耗品的上限各不相同（食物类为 2、奖券为 10 等），但作者在编辑器中无法修改。

### 修复后
物品编辑器现已显示「持有上限」字段，鼠标悬停时会给出填写建议：
- 消耗品：按需填写（如食物类常填 2~10），默认 1 表示只能持有 1 个；
- 珍视物品 / 工具：通常填 1（此类物品不叠加）。

制作出的 mod 玩家可正常使用，**玩家无需安装本插件**。

## 6. 企鹅空间编辑器增强（✅ 已增强）

### 问题
企鹅空间相关的编辑存在多处不便：
- 说说和日志混在一起没有区分，作者只能凭 ID 规律推测；
- 日志的标题在编辑器中无法看到；
- 正文和评论不分段，换行全部挤成一行；
- 发布者、评论人物、点赞者、背景音乐等许多字段仅显示数字，作者需反复查表才能确认对应对象；
- 灰色说明文字过长时会超出输入框范围。

### 修复后

**说说 / 日志区分**：左侧列表会标注类型（如 `[说说]` / `[日志]`），右侧 ID 字段也会注明类型。

**补全日志标题和浏览量**：新增「日志标题」字段（仅日志显示，说说自动隐藏）和「浏览量」字段（填 0=游戏随机，填大于 0=指定浏览量，填 -1=不显示）。

**正文分段显示**：说说、日志的正文以及评论正文，现已支持多行输入与显示，换行可正常呈现。

**数字自动翻译为文字**：以下字段原本仅显示数字，现在会在输入框下方以灰字翻译成可读内容：

| 位置 | 字段 | 显示效果示例 |
|------|------|-------------|
| 企鹅动态 | 发布者 | `对应人物：[蕾蕾（薛诗蕾）]` |
| 企鹅动态 | 自动点赞 | `对应点赞：[201=梁超杰（立即）；202=冯馨月（3秒后）]` |
| 企鹅动态 | 自带评论 | `对应评论：[30101 麦彦华：好看吗？（立即）]` |
| 企鹅动态 | 玩家评论 | `玩家评论：[30107 主角/玩家：想起它被老班没收的那天…]` |
| 企鹅评论 | 评论人物 | `对应人物：[主角/玩家，蕾蕾（薛诗蕾）]` |
| 企鹅评论 | 所回复的评论 | `根部评论：[30101 麦彦华：好看吗？]` |
| 企鹅评论 | 自带回复 | `自带回复：[30104 主角/玩家→肖清雅：一点也不恐怖（3秒后）]` |
| 企鹅评论 | 玩家回复 | `玩家回复：[30107 主角/玩家：……]` |
| 企鹅资料 | 资料人物 | `资料人物：[蕾蕾（薛诗蕾）]` |
| 企鹅资料 | 背景音乐 | `对应音乐：[42001=音乐名]`，填 `0` 显示为 `无` |

人名显示规则：填 `0` 显示为"主角/玩家"；填 `-1` 显示为"当前空间主人/对方"；其它情况优先使用空间昵称，昵称与本名不同时显示为"空间名（本名）"。

**灰字自动换行**：过长的灰色说明现已自动换行，不再超出输入框。

## 7. 送礼触发事件可编辑（✅ 已修复）

### 问题
编辑器中无法编辑「送礼触发事件」——配置列表里找不到入口，即便进入界面内部也是空白。

### 修复后
配置列表中现已加入「送礼触发事件」入口，进入后可编辑以下内容：

| 字段 | 说明 |
|------|------|
| 事件ID | 必填 |
| 物品ID | 必填，下拉列表中物品和书籍均包含（书籍带「（书）」后缀） |
| 可送礼NPC | 可赠送此礼的 NPC，多个以逗号分隔 |
| 前提条件 | 触发需满足的条件 |
| 对话ID | 按 NPC 顺序一一对应；不同 NPC 之间以分号分隔，同一 NPC 的多段对话以逗号分隔 |
| 类型标记 | 0=赠出后物品消失（已给 NPC），1=物品不消失，留空等同于 0 |
| 红点提示 | 1=显示红点提示，0=不显示（默认 1） |

左侧列表的条目名会显示为「物品名→NPC」的格式，便于辨认。

## 9. 事件关键词搜索（✅ 已修复）

### 问题
三个与事件相关的界面均缺少搜索功能：
- **事件浏览器**（从官方事件库导入事件）：仅有一个角色下拉框，无法按标题或 ID 搜索；
- **通用编辑器**（编辑自己 mod 的事件列表）：无搜索，只能滚动浏览；
- **对话编辑器**（编辑事件中的对话台词）：无搜索，只能滚动浏览。

事件浏览器动辄成百上千条（每页 40 条），缺少搜索便只能逐页翻找。

### 修复后
这三个界面均加入了搜索框，输入文字即可实时过滤：
- 搜索框为空时显示全部；
- 输入纯数字时，匹配 ID 中含该数字的条目；
- 输入文字时，匹配标题/名称/内容中含该词的条目，不区分大小写；
- 支持中文搜索；
- 新增或删除条目时会自动清空搜索，使新条目立即显示；
- 事件浏览器中「角色筛选」与「关键词搜索」可同时使用。

搜索功能仅作用于编辑器，**玩家无需安装本插件**。

## 10. 对话/选项编辑器：小游戏及胜负分支（✅ 已修复）

### 问题
对话中无法插入小游戏（辩论、砍价、打架、数独等），也无法根据小游戏的胜负接续不同的后续对话。

### 修复后
对话编辑器和选项编辑器均加入了「小游戏」输入框。填入小游戏编号后，保存时插件会自动检查配置是否合理，预览时也能看到小游戏。

#### 使用方法
1. 在对话或选项的「小游戏」输入框中填入编号（部分需附带参数）；
2. 通过下方的分支开关，设置胜利和失败分别跳转至哪句对话；
3. 保存时插件会自动检查是否存在问题并提示。

#### 保存时的检查
保存时插件会检查以下问题并弹出提示（仅提醒，不会阻止保存）：
- 该编号用于对话会导致游戏卡死（若其仅可用于选项，会提示挪至选项）；
- 需带参数的小游戏未带参数；
- 讲价、连线的参数个数不足；
- 胜负两个分支均未填写；
- 失败分支未填写（小游戏失败后游戏会卡住）；
- 同一句对话同时挂载了选项和小游戏（此时小游戏会抢占跳转，选项自身的后续将失效）。

#### 预览中的小游戏
- 预览进行到挂载小游戏的对话时，会弹出「胜利→对话X / 失败→对话Y」选择框，两条分支均可预览；
- 挂载了小游戏的选项，在预览列表中会附加「｛小游戏：名称｝」标记。

#### 附带修复一个显示问题
游戏原版存在一处显示缺陷：勾选分支开关并填好胜负跳转后，退出界面再次进入时，开关显示为未勾选（但数据仍在，游戏中分支照常触发）。此显示问题现已修复。

制作出的 mod 为普通配置文件，玩家可正常游玩，**玩家无需安装本插件**。但需注意：上述"会卡死"的情况属于游戏本身的行为，插件无法替玩家兜底，只能在编辑器侧进行检查与提示，协助作者配置出不会出错的内容。

## 11. 事件对话编辑器：人物预览由 3 个位置扩展至 9 个（✅ 已修复）

### 问题
对话编辑器下方的人物预览区分为左、中、右三个方位，每个方位实际有 3 个位置（全屏最多 9 人），但游戏原版每个方位只启用了第 1 个位置——同方位添加第 2 个人物时会被静默忽略，界面上最多只能看到 3 个人。

### 修复后
9 个位置全部可用，操作方式如下：
- 点击空位上的 **+**：为当前对话添加一个入场人物（人物将出现在你点击的位置）；
- 点击已有人物的位置：可将其替换为他人，或选择「清除」使其退场；
- 添加/替换/清除的结果会自动写入下方的「人物动作」文本框，也可继续手动编辑该文本框进行更精细的调整。

### 附带修复的几处问题
- **+ 号全部常显**：无需等前一个位置放置人物后，下一个 + 才出现；
- **点击精准落位**：原本 9 个位置的点击范围相互重叠，点击此处的 + 人物却被添加至其它方位；现已收窄点击范围，点击哪个位置人物即落在哪里；
- **清除后不错位**：原本清除一人后其他人会在屏幕上乱跳，现已按屏幕位置从左至右稳定排列，未变动的人保持原位；
- **清除后可重新添加**：原本清除一个人物后再次添加会无响应，现已修复，可反复清除与添加。

### 兼容性
- 打开、浏览、预览已有对话均只是读取，不会改动数据；
- 仅在主动点击 +、替换、清除时才会写入，写入格式与原版完全一致；
- 每方位仅 1 人的对话（最常见）显示于居中位置，与原版一致；
- 同方位 2~3 人现已全部可见，属于纯增强；
- 同方位超过 3 人（通常为手动修改配置文件制作）编辑器最多预览 3 个，但不影响数据及游戏中的实际效果。

游戏原版的对话界面本身即支持同方位多人，**玩家无需安装本插件**。

## 12. 动作指令可视化（✅ 已修复）

### 问题
编辑事件对话时缺乏可视化：
- 动作指令是一串数字（如 `1,3004,300`），难以理解每个数字的含义；
- 屏幕效果同样是一串数字；
- 为人物填入移动指令后，在编辑器中完全看不出移动距离；
- 查看动画只能借助剧情预览，而预览始终从事件第一句开始播放，调试第 40 句的动画需从头逐句点击，效率极低。

### 修复后

**动作指令悬浮翻译**：鼠标悬停于动作指令输入框上时，下方会弹出说明框，将这串数字逐条翻译为易懂的文字，例如：
> 小明：横向移动（向右300、用时0.5秒）
> 小红：淡出退场

人名会自动解析，动作参数也会翻译为"延迟x秒""站左侧"等说明。遇到无法识别的编码会显示"未知动作(编码)"，便于发现填写错误。

**屏幕效果悬浮翻译**：屏幕效果输入框同样加入了悬浮翻译，可显示"显示CG""漫画分格""屏幕震动""手机界面"等 20 余种效果的含义。

**移动位置示意**：当前对话若为某人物填入了移动指令，右侧的人物预览会直接将其移动至相应位置——填 300 移动多远、改为 500 移动多远，一目了然。切换至其它对话时人物回到默认站位。

**「预览本句」按钮**：对话编辑器右侧新增「预览本句」按钮，点击后即可直接从当前对话开始播放剧情预览，动作动画按游戏中真实的效果演出，右键关闭返回编辑器。

预览本句的特点：
- 使用的是编辑器中的最新内容——**未保存的改动亦可立即预览**；
- 从本句开始播放后，点击画面可继续向后播放，便于查看本句与后续的衔接；
- 会自动补齐开头应有的状态：本句未填背景时，会向前查找并继承背景；前面对话中入场的人物也会自动置于相应位置；
- 人物的服装、发型、姿势、朝向、放大、累计移动等状态均会向前追溯还原，使预览中的人物着装与站位与实际游玩一致。

以上功能均只影响编辑器中的显示与预览，不会写入 mod 数据，**玩家无需安装本插件**。
